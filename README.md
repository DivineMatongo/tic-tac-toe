# tic-tac-toe
In this project, the goal was to create a simple 2-player Tic Tac Toe game in order to practice creating Function Factories and encapsulated objects with private variables. I also practiced proper separation of underlying code from the user interface.

# Highlights
- The help and assistance was minimal, which really helped me practice creating a skeleton for my code from scratch, and then fleshing it out slowly.
- Creating a game, though absolutely simple and minimal, made this a very interesting project as I've always loved concepts of game design. It is as far from AAA titles as you can possibly get, but it's still fun to learn some of the underlying principles.
- This was my first project in JS where I really created and worked very closely with objects, having to plan them out properly on paper and decide how they would interact with their private variables, and which methods they would expose.

# Challenges and Lessons
- By far the biggest lesson and takeaway from this project is the sheer importance of properly planning before writing a single line of code. Coming up with a solid structure of what my objects would look like and how they would interact trivialized the rest of the work.
- Another big lesson is the importance of separating different elements of the project from each other. As advised, I first thoroughly created a working game before even thinking about the user interface. This ensured that the interface does not provide any game logic, but only serves to display it. This simplifies things as the application grows.
- One big mistake I did make is overcomplicate my objects, coming up with superflous methods that were unnecessary. Investing more time in the planning phase with pen and paper will save lots of time as you don't have to figure things out in the implementation phase.
